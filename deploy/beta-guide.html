Armor Vue
Beta Testing Guide — All Portals

What You're Testing

Armor Vue is a job management platform for window film installers. There are three separate portals, each for a different role:

⚙️ Admin Portal

Full system control — manage installers, configure settings, review flagged items. Desktop browser, email/password login.

📊 Reports Dashboard

Management view for Finance, Service, and Operations teams — job financials, service follow-ups, analytics, material orders. Desktop or tablet, email/password login.

📱 Installer Portal

Mobile-first app for field installers — daily schedule, job sign-off, photos, earnings, compliance docs. Phone browser, magic link login.

Your Access

You'll need three things before you can start. Your admin (the person who sent you this guide) needs to set these up for you first.

Admin Portal

Shared admin login — your admin will give you the email and password directly. Open on a desktop browser:

URLhttps://installer-portal-6000.web.app/index.html

Reports Dashboard

Your own staff account — admin creates it in the Admin Portal under System → Staff. You'll set your own password. Open on desktop or tablet:

URLhttps://installer-portal-6000.web.app/reports.html

Installer Portal

Magic link login — admin sends a sign-in link to your email. Open on your phone (Safari or Chrome, not Gmail's in-app browser):

URLhttps://installer-portal-6000.web.app/installer.html

Recommended Testing Order

  1. 1Start with the Admin Portal — it's the control center. Explore all four sections to understand how the system is configured before testing the downstream portals.Takes ~20 min. Desktop only.
  2. 2Move to the Reports Dashboard — log in with your own staff credentials and review the management views.Takes ~15 min. Desktop or tablet.
  3. 3Finish with the Installer Portal — experience it exactly as a field installer would, on your phone.Takes ~20 min. Phone only.
Important: Test the Admin Portal and Installer Portal in different browsers (or one in a private/incognito window). Using the same browser for both can mix up the login sessions.

Installers requesting additional pay for out-of-scope work on a job.

Getting In

The installer portal uses magic link login — no password. The admin provisions your account and the system emails you a one-time sign-in link.

  1. 1Admin provisions your account in the Admin Portal → Installers → Provision.You'll receive an email with a "Sign in to installer-portal-6000" link.
  2. 2Open the email on your phone. If using Gmail, tap ⋮ → "Open in Chrome" or "Open in Safari" — don't use Gmail's built-in browser.The link is single-use. Opening it in the wrong browser wastes it.
  3. 3A confirmation screen appears. Tap Sign In to Portal → to proceed.
Link expired? Ask the admin to resend via the ⟳ Link button next to your name in the Installers section.

How to Report Issues

For every issue you find, capture these five things:

  1. 1Portal — Admin, Reports, or Installer
  2. 2What you were doing — which section/tab, what action you took
  3. 3What you expected — what should have happened
  4. 4What actually happened — error message, blank screen, wrong data
  5. 5Screenshot — always helpful, especially for layout issuesDevice + browser: e.g., "iPhone 15 / Safari" or "Windows Chrome"

What We're Specifically Looking For

🐛 Bugs

Anything that errors, crashes, shows a blank screen, or doesn't save correctly.

📱 Mobile Experience

Buttons too small to tap, text overflowing, keyboards covering inputs, slow loading on cellular.

🤔 Confusing Flows

Anything that's unclear about what to do next, missing labels, ambiguous buttons.

🔒 Data / Privacy

Any place where an installer can see data they shouldn't (especially financial data).

Things That Are Intentional

Installers never see the full job contract amount — only their own pay portion. This is by design.
The magic link is single-use and expires. If it doesn't work, the admin resends a fresh one.
The tutorial only runs on first login. It can be replayed from the profile menu — it won't re-run automatically.
The admin portal only works in one browser at a time with the admin account. Testing the installer portal in the same browser will log out the admin session.
